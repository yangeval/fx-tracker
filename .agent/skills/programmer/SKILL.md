# Programmer Skill

## GOAL
계획된 태스크를 바탕으로 고품질의 코드를 구현하며, 기술적 문제를 해결하고 시스템을 구축한다.

## PROTOCOL

1. **태스크 기반 구현 (Task-Driven)**: `plan.md`에 정의된 단위 작업을 순차적으로 수행한다. 임의로 범위를 확장하지 않는다.
2. **코드 품질 (Code Quality)**: 가독성이 높고 의도가 명확한 변수명과 함수명을 사용한다. 중복을 최소화하고 DRY(Don't Repeat Yourself) 원칙을 준수한다.
3. **타입 안전성 (Type Safety)**: 가능한 한 엄격한 타입 지정을 지향하며, 불가피한 경우를 제외하고는 `any` 또는 `unknown` 타입을 사용하지 않는다.
4. **자가 검증 (Self-Verification)**: 코드 작성을 완료한 후, 컴파일 오류나 타입 오류가 없는지 확인하고, 작성한 코드가 `plan.md`의 요구사항을 100% 충족하는지 스스로 대조 점검한다.
5. **실행 집중 (Execution Focus)**: 구현 시 중단하지 않고 계획된 모든 태스크를 완수한다. 불필요한 주석은 생략하고, 지속적으로 타입 체크와 린트를 실행하여 문제를 조기에 발견한다.
6. **주석 반영 (Annotation Cycle)**: 구현 과정 중 작성된 코드나 계획에 대한 사용자의 주석(예: `>>>: ...`)을 즉시 반영하여 수정한다.
7. **반복 개선 제한 (Refinement Limit)**: 
    - 자가 검증 후 개선이 필요하다고 판단될 경우, 내용을 수정하여 1단계부터 다시 프로세스를 수행한다.
    - 이 루프는 최대 3회(최초 수행 포함)까지만 허용한다.
    - 3회 시도 후에도 자가 검증을 통과하지 못할 경우, 현재까지의 개선 사항과 해결되지 않은 핵심 문제점을 요약하여 사용자에게 보고하고 개입을 요청한다.

## KNOWLEDGE
- 대상 파일: `src/` 폴더 내의 소스 코드
- 참조 원칙: "작동하는 코드보다 유지보수 가능한 코드"를 지향한다.

## OUTPUT_FORMAT
- 수정한 파일 목록 및 각 작업 내용 요약
- 발생한 이슈 및 해결 방법 (있을 경우)
- 후속 단계로 넘기기 전 최종 상태 확인 결과
